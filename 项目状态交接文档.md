## 1. 产品背景与评分模型

改造依据飞书文档《服务评估 - 新标准梳理》(`docx/SGtsdBfNyo6Z61xWV10cp4MNnNc`)，本地副本：`服务评估_新标准梳理_原文完整复制.md`。

**评分模型（6 + 1 结构）：**
- **SQS（Service Quality Score，服务质量分）** = 6 维等权平均：Understanding Accuracy / Execution Correctness / Solution Adoption / Responsiveness / Service Efficiency / Language Quality。
- **UES（User Experience Score，用户预期达成 / 用户满意分的一维）** = 1 维（Service Outcome Expectation）。旧称 UEF，已全局改回 UES。
- **User Satisfaction（北极星）** = 归一化加权 `w(SQS)·SQS + w(UES)·UES`（权重可在 Settings 编辑，默认均分）。
- **Gatekeeper / 一票否决机制已彻底删除**（文档最新版删除，代码全部移除）。
- **Responsiveness（响应及时性）** 由系统按时间戳自动判定，标注页只读、不参与人工打分。

## 2. 核心角色模型（动态角色 + 防自审）

角色不预先指定，由"谁先动了这条 case"动态决定：
1. **A（第一评标注员）**：第一个评某条 case 的人。
2. **B（Reviewer / Back-to-Back）**：任何**不是这条 A** 的人都可以做二次评（团队灵活分配）。有人二次评并提交 → 该 case 成为 **Double-blind**；没人二评 → **Normal**。
3. **C（QC）**：既没做过这条的 A、也没做过它的 B 的第三人，才能做 QC 复核（overwrite 定案）。

**防自审**：A 不能对自己评过的做 B/C；B 不能 QC 自己 B2B 过的（交给 C）。

**关键判定链（单一事实源）：**
- 进 QC 门槛 = **A 已提交**（`aResultStatus === "Submitted"`）。
- 双盲 vs Normal = **B 是否已提交**（`bResultStatus === "Submitted"`），全项目统一用这个判定。
- 已定案 = **C overwrite**（`currentState === "Final Result Ready"`）。

## 4. 各页面当前实现要点（= 交互描述当前版）

### 首页
- Old / New Rule 切换（彻底隔离）；Source type、Task type 筛选。
- 实时统计列：Cases、Assigned（≠Unassigned 计数）、Completed（A评完及之后）、SQS Avg、UES Avg、User Satisfaction Avg、SQS Pass Rate、QC Accuracy（= C 定案里 C 与 A 打分完全一致的占比，无定案 "—"）。**没有 Standard Version 列**。
- 每行：Detail / Assign / Audit / Export to ByteHi / Download；右上：Import from ByteHi / Upload CSV / Settings。
- **Assign（Distribute）**：只分 A 一轮，多个 QA + quantity 按顺序**递归切分**未分配 case；超额报错；确认后状态 `A Assigned`、qaOwner=该 A（不是操作者）。

### Detail
- 列：Session ID、Subtype、Knowledge Source、SQS（可展开）、UES（可展开）、User Satisfaction、Transfer to human（只读 Yes/No）、Assign QA、Status、Actions、Latest Activity Log。
- Filter：Service Subtype / Knowledge Source / Problem Type(R1/R2/R3) / SQS(Pass/No Pass) / Human Result(Has Human/Bot Only)。
- 点 Session ID → 只读 View。
- **单条 Assign QA**：Unassigned → 指派进 A 流程；已在流程中 → 只改负责人标签、不打回、不覆盖已记录 A。
- **Actions（动态解析）**：A 未评→Annotate；A 已评且本人→Annotate(可改)；A 已评且非本人→Review(B2B)；已 B2B完/已定案→View。防自审拦截。
- **Status 全集**：Unassigned / A Assigned / Completed Annotation / Back-to-Back Completed / Selected for C QC / Final Result Ready。

### Annotation 评分页
- 左 evidence 60% / 右评分 40%；Chatbot、Ticket 同一套会话窗口。
- 顶部总分预览，右上 Role 徽标；直接点 0/1/2/3（分数与 reason 绑定）；Responsiveness 只读。
- A/B 盲评不预填；C 预填上一版（B→A）并 overwrite，提交即 Final。
- View / 冻结时只读；Human Result 仅转人工时出现。

### QC / Audit
- Audit 唯一入口；A 评完即进可抽样范围；双盲/Normal 由"B 是否提交"判定，都能在 QC 看到。
- 概览：Completed Annotations / Sampled for C / Final Ready；按钮「Set Sampling」「View Completed Results」（无可抽 case 时 Set Sampling 置灰）。
- **Set Sampling**：顶部 "Available to sample: X of Y"；scope=All QAs / By QA（下拉只列还有未抽 case 的人，A 或 B 参与都算）；method=Percentage(1-100) / Absolute；预估口径与实际一致；**抽样累加**（不退回已抽、只从未抽池抽）。
- **C Review**：A/B/C Review Summary 三列对比（A列=A评人、B列=B评人、C列=当前用户）；双盲差异红底 Δ、C 覆盖标蓝；C 提交 → Final Result Ready。
- 最终结果：定案用 C 结果，未抽中用 A 结果；A/B/C 各版本保留。

### 其他
- **Activity Log**：Timestamp(业务事件) vs Activity Log(平台操作流水) 区分；Detail 只展示最近一条；抽屉看完整流水 + Export CSV。
- **Submit & Export**：`annotation_summary.csv`（汇总层）、`annotation_data.csv`（明细层）。
- **评分标准版本管理**：结果记录提交版本，历史不自动重算/静默覆盖。
- **空白起点**：session 是工作项，初始全未分配、无评分、无日志。

## 5. 已完成的改造历程（按批次）

- **第一批**：UEF→UES 全局改名；真实 20-40 个 session；Batch Edit 刷新修复；Transfer to human 改系统识别只读；SQS/UES 汇总列可展开。
- **第二批（Audit Portal）**：动态 A/B/C 角色；提交即锁定；C 覆盖即为准；基础防自审；升级 CReview（diff 高亮）而非新建页。
- **第三批（blank 起点）**：撤掉所有 case/日志；全未分配起点；QC Accuracy 清零；删除 QC Status 列/类型。
- **后续增量修复**：
  - 分配递归切分 + 超额报错；Assign QA 列显示正确的人（qaOwner=A 而非操作者）。
  - 首页 Assigned / Completed 分列 + 分数列实时聚合 + QC Accuracy 实时（C vs A 一致率）。
  - 「Review」文案（原 B Review）；canDoB 放开到"非 A 即可 Review"。
  - 双盲判定统一为 bResultStatus；去掉 "Waiting B" 死状态；A-submit 不再读 backToBackEnabled。
  - 无 role 打开 = 只读 View，doSubmit 绝不隐式 A（防覆盖）。
  - 防自审文案按 role；批量编辑排除已定案、复选框置灰。
  - assignSingleQa 不再把已进流程的 case 打回 A 阶段。
  - Sampling：预估口径与实际一致、抽样累加、By QA 含 A/B、poolCount 控制按钮禁用、百分比限 1-100、关闭重置。
  - Activity Log 抽屉加 Export CSV。
  - Distribute 弹窗删掉「A 轮 / AB 两轮」，改为直接「Annotators」。
