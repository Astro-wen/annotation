## Intro & Goal

### What are we building?

本项目旨在升级 ByteHi Manual Annotation Tool，以适配最新的客服服务评估标准：整体北极星指标为**用户满意分（User Experience Score）**，由**服务质量分（Service Quality Score，SQS）** 与**用户预期达成（User Expectation Fulfillment，UES）** 共同构成。工具侧支持按当前生效的评分标准版本进行标注、展示、导出与历史结果追溯。

核心改动点包括：

1. **规则隔离**：在任务列表及详情页实现新旧规则的完全隔离，避免污染（首页顶部一个 Old Rule / New Rule 开关）。
2. **打分架构重构 [NEW]**：将原有的 P/Q/I 架构替换为最新版用户满意分框架——服务质量分（SQS）包含理解准确性、执行正确性、方案采纳、响应及时性、服务效率、语言质量 6 个维度；用户预期达成作为独立用户视角评分；最终用户满意分按当前生效的标准版本计算。
3. **标准版本与自动化辅助逻辑 [NEW]**：工具不内置旧版一票归零 / Gatekeeper 规则。评分字段、分档、权重、自动计算与校验逻辑均由当前生效的评分标准版本驱动。系统可提供负向信号提示等辅助能力，但不替代人工确认，也不在标准未定义的情况下自动将其他维度归零。其中响应及时性（Responsiveness）按最新标准由系统根据响应时间戳自动判定，跳过该维度的人工标注（展示为只读，不参与人工打分 / 编辑）。
4. **标注 / QC 流程与角色 [NEW]**：分配即定模式——Normal 单人评，或 Back-to-Back 双人盲评（分配时一行填 A、B 和数量，预先配对）。QC 由第三人 C 在 Audit 里对抽样的 case 复核、overwrite 定案；A / B / C 三个角色由分配和行为决定，系统自带防自审（同一条 case 上 A 和 B 不能同人，C 不能是评过的 A 或 B）。团队可以在「双人互检爬坡」和「单人 + 抽检放量」两阶段之间无缝切换，工具不锁死分工。
5. **可追溯 [NEW]**：每条结果记录提交时的评分标准版本；每次分配、标注、修改、复核都进 Activity Log，点版本号可回看当时冻结的评分快照。
6. **上下文增强 [NEW]（部分为预期设计）**：以对话（Evidence）为核心展示会话证据；后续集成 Skill Flow、变量值及用户反馈信号，支撑标注人精准判定——目前已落地会话证据展示，Skill Flow / 变量值等尚未真正接入。

### Why build it?

1. **对齐集团服务评估新标准**：现有的 GE Rate（优良率）体系已无法满足对智能 / 人工服务进行「平台标准执行」与「用户主观体验」的双重精细化评估需求。
2. **作为 AI Evaluation 的 Ground Truth 生产线**：人工标注结果是衡量 AI 自动评估准确率的基准。工具必须提供结构化、可审计的标注产出（本工具的 Activity Log、版本快照、CSV 导出即服务于此）。
3. **提升标注效率与一致性**：通过 Score-first 的引导式交互（分数与 reason 直接点选，无多层下拉）及内置 Rubric 助手，降低标注人的心智负担，减少人为判分误差。
